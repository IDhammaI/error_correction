# 前端开发规范

> 本文档从 `CLAUDE.md` 中提取并整理，作为前端开发的完整参考手册。

---

## 1. 技术栈

| 类别 | 选型 | 备注 |
|------|------|------|
| 框架 | Vue 3 + `<script setup>` | Composition API，禁止 Options API |
| 构建 | Vite | 多页面入口：`index.html`（介绍页）+ `app.html`（工作台） |
| 样式 | Tailwind CSS 3 | utility-first，class-based dark mode |
| 交互组件 | HeadlessUI Vue | 用于可访问的弹窗、下拉等 |
| HTML 净化 | DOMPurify | 白名单在 `utils.js` 的 `ALLOWED_HTML_TAGS` |
| 语言 | 纯 JavaScript | 不使用 TypeScript |
| 网络请求 | 原生 fetch | 不使用 axios，文件上传用 XHR |
| 状态管理 | 无 | 全局状态集中在 `App.vue`（ref/reactive/computed/watch） |
| CDN 依赖 | Font Awesome 6.5、MathJax 3、SortableJS 1.15、Chart.js 4 | 非 npm 安装 |

## 2. 布局规范

### PC 端
- 左侧固定侧边栏：`aside w-64`
- 右侧主内容区：flex-1

### 移动端
- 底部 Tab 导航栏：`fixed bottom-0`
- 全宽内容区

### 视图切换
- `currentView` ref 控制视图（`v-show` 切换，非 `v-if`）
- 路由映射在 `VIEW_TO_PATH` 对象中集中管理

## 3. 间距节奏（8pt Grid）

**所有 padding / margin / gap 只允许使用 4 的倍数**，对应 Tailwind 档位：

| 用途 | 允许值（Tailwind） | 像素 |
|------|-------------------|------|
| 微间距（图标、标签内边距） | `p-1` `p-2` `gap-1` `gap-2` | 4px、8px |
| 小间距（按钮、行内元素） | `p-3` `p-4` `gap-3` `gap-4` | 12px、16px |
| 中间距（卡片内边距、区块间距） | `p-6` `p-8` `gap-6` `gap-8` | 24px、32px |
| 大间距（页面边距、区域分割） | `p-10` `p-12` `gap-10` `gap-12` | 40px、48px |

### 禁止使用

```
❌ p-5  p-7  p-9  gap-5  gap-7  gap-9
❌ px-[17px]  py-[13px]  gap-[22px]  （任意魔法数字）
❌ space-y-3  space-y-5  space-y-7  （非4倍数的 space 值）
```

### 正确示例

```html
<!-- ✅ 正确 -->
<div class="p-6 gap-4">
<div class="mb-8 px-4 py-2">

<!-- ❌ 错误 -->
<div class="p-5 gap-3">
<div class="mb-7 px-[15px]">
```

## 4. 排版层级（Typography）

字号只允许以下 **7 档**，禁止使用其他任何值：

| 层级 | Tailwind | 像素 | 行高 | 用途 |
|------|----------|------|------|------|
| 超大标题 | `text-4xl` | 36px | `leading-tight`（1.25） | 落地页 Hero、各区块主标题 |
| 大标题 | `text-3xl` | 30px | `leading-tight`（1.25） | 页面主标题、工作台区块标题 |
| 标题 | `text-2xl` | 24px | `leading-snug`（1.375） | 区块标题、Modal 标题 |
| 副标题 | `text-xl` | 20px | `leading-snug` | 卡片标题、侧边栏项 |
| 正文 | `text-base` | 16px | `leading-relaxed`（1.625） | 题目内容、说明文字 |
| 辅助 | `text-sm` | 14px | `leading-relaxed` | 标签、次要信息 |
| 标注 | `text-xs` | 12px | `leading-normal`（1.5） | 元数据、时间戳、徽标 |

### 禁止使用

```
❌ text-lg  text-[10px]  text-[11px]  text-[13px]  text-[15px]
❌ 任何 text-[Npx] 自定义值
```

### 信息密度原则

每张 Card 只允许 **1 个主信息 + 2 个辅信息**，禁止堆砌超过 3 层文字层级。

## 5. 配色 Token

**只允许以下色系，禁止随意引入彩虹色。**

### 语义色

| 语义 | 亮色 | 暗色 |
|------|------|------|
| 页面背景 | `white` / `slate-50` | `slate-900` / `slate-950` / `[#0A0A0F]` |
| 卡片面（玻璃态） | `bg-white/70 backdrop-blur-xl` | `bg-white/[0.03] backdrop-blur-xl` |
| 边框 | `border-slate-200/60` | `border-white/10` |
| 主操作 | `blue-600` / `indigo-600` | `indigo-500` |
| 成功 | `emerald-600` | `emerald-400` |
| 错误 | `rose-600` | `rose-400` |
| 中性文字 | `slate-900` / `slate-700` / `slate-500` / `slate-400` | `white` / `slate-200` / `slate-400` / `slate-500` |

### 禁止使用的颜色

```
❌ orange  amber  fuchsia  pink  cyan  teal  lime  yellow  violet  purple
```

> 如果需要状态区分（如"复习中"），使用 `indigo`（主色）或 `slate`（中性色），不要用 amber/orange。

### 玻璃态卡片标准写法

```html
<!-- 亮色 -->
<div class="rounded-2xl border border-slate-200/60 bg-white/70 p-6 shadow-sm backdrop-blur-xl">

<!-- 暗色 -->
<div class="dark:border-white/10 dark:bg-white/[0.03]">
```

## 6. 圆角 Token

| 元素 | 值 | 示例 |
|------|-----|------|
| 大卡片、Modal、弹窗 | `rounded-2xl` / `rounded-3xl` | 内容容器、对话框 |
| 按钮、输入框、小卡片 | `rounded-xl` | 操作按钮、表单控件 |
| 标签、Badge、Pill | `rounded-full` | 学科标签、状态徽标、数量指示器 |
| 图标容器、小元素 | `rounded-lg` | 复选框、头像容器 |

### 禁止使用

```
❌ rounded-[2rem]  rounded-md  rounded-sm  rounded-none
❌ 任何 rounded-[Npx] 自定义值
```

---

## 7. 阴影层级

**只允许 2 层，禁止滥用大投影。**

| 状态 | 值 | 用途 |
|------|-----|------|
| 默认 | `shadow-sm` | 卡片、输入框的静息态 |
| Hover / 激活 | `shadow-md` 或品牌色光晕 `shadow-blue-500/20` | 交互反馈 |

### 禁止使用

```
❌ shadow-lg  shadow-xl  shadow-2xl  shadow-inner
```

## 8. 按钮样式

### 尺寸

- 高度统一 `h-10`（40px）
- 图标尺寸 `size-4`（16px）或 `size-5`（20px）

### 类型

| 类型 | 样式 |
|------|------|
| 主按钮 | `h-10 rounded-xl bg-blue-600 text-white text-sm font-bold` + hover 光晕 |
| 成功按钮 | `h-10 rounded-xl border border-emerald-500/30 bg-emerald-50/80 text-emerald-700 text-sm font-bold` |
| 次按钮 | `h-10 rounded-xl border border-slate-200/60 bg-white/60 text-slate-700 text-sm font-bold` |

### 通用属性

```html
<!-- 每个按钮都必须包含 -->
class="inline-flex items-center justify-center gap-2 transition-all"

<!-- 禁用态 -->
class="disabled:cursor-not-allowed disabled:opacity-50"
```

### 禁止

```
❌ h-11  h-12  h-9  h-8  （非标准高度）
❌ hover:-translate-y-0.5  （按钮不应上移）
```

## 9. 暗色模式

- **所有元素必须包含 `dark:` 变体**，不能只写亮色样式
- 暗色背景使用 `slate-900` / `slate-950` / `[#0A0A0F]`
- 主题切换支持 View Transitions API 圆形扩散动画
- `localStorage.theme` 持久化用户偏好

### 暗色模式检查清单

```
✅ 背景色有 dark: 变体
✅ 文字颜色有 dark: 变体
✅ 边框颜色有 dark: 变体
✅ hover 状态有 dark: 变体
✅ focus 状态有 dark: 变体
```

## 10. 组件规范

### 文件组织

- 新功能提取为独立 `.vue` 组件放 `src/components/`
- 关键 DOM 元素添加语义类名或 `data-testid`
- 父子通信：props 向下，emits 向上
- UI 文案使用中文
- API 路径前缀 `/api/`

### 已抽离的公共组件

| 组件 | 用途 |
|------|------|
| `GlassCard` | 玻璃态卡片容器，支持 `padding` `rounded` props |
| `PageHeader` | 页面标题区，支持徽章、标题、副标题、`#extra` 插槽 |
| `StatCard` | 统计数字卡片，支持 `label` `icon` `value` `unit` `color` |
| `QuestionItem` | 题目列表项，支持选择、状态显示、`#extra` `#actions` 插槽 |
| `ProviderSection` | API Provider 配置区块（标题 + 列表项 + 空状态） |
| `SearchInput` | 搜索输入框，支持 `v-model` `label` `placeholder` `icon` |
| `CustomSelect` | 下拉选择框（玻璃态） |
| `CalendarPicker` | 日期选择器 |
| `SplitLoading` | AI 分割任务全屏遮罩动画 |

### 组件 props 命名

- 布尔值用 `is` / `show` / `has` 前缀：`showStatus` `selectable`
- 事件用动词：`@click` `@toggle-select` `@remove`
- 颜色类 prop 使用 Tailwind 颜色名：`color="indigo"`

## 11. 交互规范

### Hover 效果

- 卡片：`hover:shadow-md`（阴影加深，不位移）
- 按钮：颜色/透明度变化 + `transition-all`
- **禁止** `hover:-translate-y` 上移效果（除非是落地页动画）

### 过渡动画

- 所有可交互元素必须包含 `transition-all` 或具体的 transition 属性
- 视图切换使用 `<Transition name="view-fade">`
- 弹窗使用 HeadlessUI 内置过渡

### 下拉面板

- 面板背景不透明：亮色 `bg-white`，暗色 `bg-slate-900`
- 避免透明穿透导致文字重叠

## 12. 快速自检清单

开发新组件或修改现有组件时，逐项检查：

- [ ] 间距是否为 4 的倍数（无 p-5、gap-5 等）
- [ ] 字号是否在 7 档之内（无 text-[10px] 等）
- [ ] 配色是否在允许色系内（无 orange、amber 等）
- [ ] 圆角是否符合 Token（标签用 rounded-full，卡片用 rounded-2xl）
- [ ] 阴影是否只用 shadow-sm / shadow-md
- [ ] 按钮高度是否为 h-10
- [ ] 所有样式是否有 dark: 变体
- [ ] 是否有不必要的 hover 位移
- [ ] 是否有 transition-all 或过渡动画
- [ ] Card 信息层级是否不超过 3 层
