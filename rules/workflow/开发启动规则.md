# 开发启动规则

## 后端启动

基础启动：

```bash
cd backend
python web_app.py
```

如果使用 conda 环境，当前确认过的环境名是 `error_correction`：

```powershell
cd backend
conda activate error_correction
python .\web_app.py
```

如果当前 PowerShell 没有初始化 conda，可以在项目根目录执行：

```powershell
conda run -n error_correction python .\backend\web_app.py
```

后端 Flask API 默认监听 `localhost:5001`。

`backend/web_app.py` 是纯 API 服务，不托管前端页面。直接访问后端端口的前端路由会得到 JSON 404。

## 前端启动

```bash
cd frontend
npm run dev
```

前端通过 Vite dev server 访问，并代理 `/api` 等请求到后端。

前端开发服务默认访问地址：

```text
http://127.0.0.1:5173
```

## 一次性启动两个 PowerShell 窗口

使用已初始化 conda 的 PowerShell：

```powershell
$root = (Get-Location).Path
Start-Process powershell -ArgumentList '-NoExit','-Command',"Set-Location '$root\backend'; conda activate error_correction; python .\web_app.py"
Start-Process powershell -ArgumentList '-NoExit','-Command',"Set-Location '$root\frontend'; npm run dev"
```

如果当前 PowerShell 没有初始化 conda，后端窗口改用 `conda run`：

```powershell
$root = (Get-Location).Path
Start-Process powershell -ArgumentList '-NoExit','-Command',"Set-Location '$root'; conda run -n error_correction python .\backend\web_app.py"
Start-Process powershell -ArgumentList '-NoExit','-Command',"Set-Location '$root\frontend'; npm run dev"
```

## 端口检查

检查后端是否监听：

```powershell
Get-NetTCPConnection -LocalPort 5001 -State Listen
```

检查前端是否监听：

```powershell
Get-NetTCPConnection -LocalPort 5173 -State Listen
```
