# 前端规则

## 技术规则

- 使用 Vue 3、`<script setup>` 和 Composition API。
- 不使用 Options API。
- 使用 Vite、Vue Router 4、Tailwind CSS、HeadlessUI 和 DOMPurify。
- 当前项目使用 JavaScript，不使用 TypeScript。
- API 调用集中放在 `frontend/src/api/`。
- 文件上传保留 XHR 或现有上传工具，以支持进度回调。
- UI 文案使用中文。
- API 路径使用 `/api/` 前缀。
- 新功能优先拆成独立组件或 composable，遵循现有目录结构。
- 认证、主题、工作台、对话等共享状态优先复用现有 composable。
- Modal、Dropdown 等浮层优先使用现有组件和 `useClickOutside`。
- 渲染 Markdown 后需要触发 MathJax 公式排版。
- HTML 内容必须经过 DOMPurify 或现有净化工具处理。

## 设计规则

- 优先遵循现有基础组件和全局样式。
- 新 UI 要兼容暗色和亮色主题。
- 常用间距优先使用 Tailwind 标准尺度。
- 不新增难以维护的魔法数字样式。
- 不为了局部效果绕开已有设计系统。
- 交互组件需要考虑禁用、加载、错误和空状态。
