# 前端规则

## 技术栈与入口

- 使用 Vue 3、`<script setup>` 和 Composition API。
- 不使用 Options API。
- 当前项目使用 JavaScript，不使用 TypeScript。
- 使用 Vite 7、Vue Router 4、Tailwind CSS、HeadlessUI 和 DOMPurify。
- 图表、导出等能力优先复用现有依赖，如 `echarts`、`html2canvas`、`jspdf`。
- 前端 SPA 入口是 `frontend/app.html`，不是默认的 `index.html`。
- Vite 已配置 `@` 指向 `frontend/src/`，模块导入优先使用该别名。

## 目录分层

- 页面和布局放在 `frontend/src/views/`，按 `home`、`auth`、`app` 分组。
- 通用基础组件放在 `frontend/src/components/base/`，可复用组件优先沉淀到这里。
- 业务组件放在 `frontend/src/components/features/`，按功能域继续分组，如 `app/workspace`、`app/notes`。
- 共享状态和可复用逻辑放在 `frontend/src/composables/`，认证、主题、导航、上传、Toast 等优先复用现有 composable。
- API 调用集中放在 `frontend/src/api/`，不要在组件里直接散落写 `fetch`。
- 纯函数和渲染辅助逻辑放在 `frontend/src/utils/`。
- 前端测试放在 `frontend/src/__tests__/`，沿用现有 `Vitest + jsdom` 结构。

## 路由与页面组织

- 路由统一维护在 `frontend/src/router/index.js`。
- 当前页面分为三类布局：`home`、`auth`、`app`。
- 工作台主路由使用 `/app/:view?/:subview?`，由 `AppLayout` 负责切换具体子视图。
- 新增页面时，优先判断是新增 `view`、新增现有 `view` 的子面板，还是新增 `features` 业务组件，不要把大量业务逻辑直接堆进路由文件或布局文件。
- 路由鉴权、跨布局切换动画等全局行为应继续放在路由守卫和现有 composable 中统一处理。

## 开发规则

- API 路径使用相对路径和现有代理配置，业务接口保持 `/api/` 前缀。
- 请求封装优先复用 `frontend/src/api/client.js` 的统一错误处理。
- 文件上传保留 XHR 或现有上传工具，以支持进度回调。
- 新功能优先拆成独立组件或 composable，遵循现有目录结构。
- Toast、主题、工作台导航、认证状态等全局能力优先复用 `useToast`、`useWorkspaceToast`、`useTheme`、`useAuth`、`useWorkspaceNav` 等现有实现。
- 可复用的弹窗、下拉、Popover、图片预览等交互优先复用 `components/base/` 和现有 overlay 相关 composable。
- 新增通用基础组件时，同步维护 `frontend/src/components/base/index.js`，必要时补充 `registry.js`。
- UI 文案使用中文。
- HTML 内容必须经过 DOMPurify 或现有净化工具处理。
- 渲染 Markdown 后需要触发 MathJax 公式排版。

## 设计与样式规则
- 当前项目使用 JavaScript，不使用 TypeScript。
- 新 UI 要兼容暗色和亮色主题。
- 常用间距优先使用 Tailwind 标准尺度。
- 主题色通过现有 CSS 变量和 `useTheme` 管理，不要各处自造一套颜色状态。
- 不新增难以维护的魔法数字样式。
- 不为了局部效果绕开已有设计系统。
- 交互组件需要考虑禁用、加载、错误和空状态。
- 交互组件需要考虑禁用、加载、错误和空状态。

## 测试规则

- 使用 `Vitest`、`jsdom` 和 `@vue/test-utils`。
- 新增测试优先覆盖高价值逻辑，如 `utils`、`composables`、`api` 封装和关键交互状态。
- 测试描述使用中文，与现有测试风格保持一致。
- mock 和副作用清理沿用现有模式，避免把测试写成大而脆弱的页面快照。
